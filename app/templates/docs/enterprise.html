enterprise

Enterprises are resources that bring together a set of organizations and members into a single collection. The enterprise resource makes heavy use of the SCIM API documented here. For a better understanding of the schema for enterprises, read the overview provided here.

  • Required permissions: read

GET /1/enterprises/[idEnterprise]/members

  • Required permissions: read
  • Arguments

    • sortBy (optional)
      • Default: none
      • Valid Values:

        This parameter expects a SCIM-style sorting value.

        Note that the members array returned will be paginated if `members` is 'normal' or 'admins'. Pagination can be controlled with member_startIndex, etc, but the API response will not contain the total available result count or pagination status data.

    • sortOrder (optional)
      • Default: none
      • Valid Values:
        • ascendind
        • descending
        • asc
        • desc
    • startIndex (optional)
      • Default: none
      • Valid Values:

        Any integer between 0 and 9999.

    • count (optional)
      • Default: none
      • Valid Values:

        • SCIM-style filter.

    • organization_fields (optional)
      • Default: displayName
      • Valid Values:

    • board_fields (optional)

GET /1/enterprises/[idEnterprise]/transferrable/organization/[idorganization]

Used to determine if an organization can be transferred into the enterprise; if not — returns the untransferred members blocking that, if so — reports the number of net new enterprise members it will result in.

  • Required permissions: write
  • Arguments
    • idOrganization (required)
      • Default: none
      • Valid Values:

        An ID of a Board resource.
